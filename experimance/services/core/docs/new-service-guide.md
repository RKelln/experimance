# New Core Service Guide

This guide walks through creating a new core service for the Experimance project ecosystem, following the same patterns as `experimance_core` and `fire_core`.

A core service is the central orchestrator of an interactive art installation. It typically:
- Receives input from sensors or an agent service.
- Processes input with LLMs or custom logic.
- Generates image prompts and coordinates with image generation services.
- Manages state transitions and drives visual output.

## Prerequisites

- Python async/await familiarity.
- ZeroMQ message patterns (PUB/SUB, PUSH/PULL). See [architecture.md](architecture.md).
- Pydantic models and configuration management.
- The Experimance project structure (see `libs/common/README*.md`).

---

## Step 1 – Project Directory

Create `projects/your_project_name/` with:

```
projects/your_project_name/
├── .env               # OPENAI_API_KEY etc.
├── config.toml        # Project-level config
├── core.toml          # Core service config
├── constants.py       # Project-specific constants (optional)
├── schemas.py         # Project-specific message schemas
└── schemas.pyi        # Type stubs (generated by scripts/update_pyi_stubs.py)
```

## Step 2 – Project Schemas

**`projects/your_project_name/schemas.py`**

```python
from enum import Enum
from typing import Optional
from experimance_common.schemas_base import (
    StringComparableEnum,
    SpaceTimeUpdate as _BaseSpaceTimeUpdate,
    RenderRequest as _BaseRenderRequest,
    ImageReady as _BaseImageReady,
    DisplayMedia as _BaseDisplayMedia,
    MessageBase,
    MessageType as _BaseMessageType,
    ContentType,
)

class YourEnum(StringComparableEnum):
    VALUE1 = "value1"
    VALUE2 = "value2"

# Python enums cannot be extended; redeclare the entire enum.
class MessageType(StringComparableEnum):
    # ---- copy all base types ----
    SPACE_TIME_UPDATE = "SpaceTimeUpdate"
    RENDER_REQUEST = "RenderRequest"
    IMAGE_READY = "ImageReady"
    DISPLAY_MEDIA = "DisplayMedia"
    PRESENCE_STATUS = "PresenceStatus"
    # ... (copy remaining base types from experimance_common.schemas_base)

    # ---- your additions ----
    YOUR_NEW_MESSAGE = "YourNewMessage"

class RenderRequest(_BaseRenderRequest):
    """Optional project-specific RenderRequest fields."""
    your_field: Optional[str] = None

class YourNewMessage(MessageBase):
    type: MessageType = MessageType.YOUR_NEW_MESSAGE
    content: str
    timestamp: Optional[str] = None
```

**Key rules:**
- Always import base classes from `experimance_common.schemas_base`.
- Redeclare `MessageType` completely – do not try to extend the base enum.
- Keep `DisplayMedia` unchanged unless you have a strong reason to extend it.

## Step 3 – Service Configuration

**`services/core/src/your_project_core/config.py`**

```python
from pydantic import BaseModel, Field
from experimance_common.config import BaseServiceConfig
from experimance_common.constants import DEFAULT_PORTS, ZMQ_TCP_BIND_PREFIX, ZMQ_TCP_CONNECT_PREFIX
from experimance_common.zmq.config import (
    ControllerServiceConfig, WorkerConfig,
    PublisherConfig, SubscriberConfig,
    ControllerPushConfig, ControllerPullConfig, MessageType,
)

class YourProjectConfig(BaseModel):
    param1: str = Field(default="default_value")
    param2: int = Field(default=42)

class YourProjectCoreConfig(BaseServiceConfig):
    service_name: str = Field(default="your_project_core")

    your_project: YourProjectConfig = Field(default_factory=YourProjectConfig)

    zmq: ControllerServiceConfig = Field(
        default_factory=lambda: ControllerServiceConfig(
            name="your-project-core",
            publisher=PublisherConfig(
                address=ZMQ_TCP_BIND_PREFIX,
                port=DEFAULT_PORTS["events"],
                default_topic=MessageType.DISPLAY_MEDIA,
            ),
            subscriber=SubscriberConfig(
                address=ZMQ_TCP_CONNECT_PREFIX,
                port=DEFAULT_PORTS["agent"],
                topics=[MessageType.YOUR_NEW_MESSAGE],
            ),
            workers={
                "image_server": WorkerConfig(
                    name="image_server",
                    push_config=ControllerPushConfig(port=DEFAULT_PORTS["image_requests"]),
                    pull_config=ControllerPullConfig(port=DEFAULT_PORTS["image_results"]),
                ),
            },
        )
    )
```

**Tips:**
- Use `DEFAULT_PORTS` constants rather than hard-coded port numbers.
- Use `"image_server"` as the worker name – that is the name the image generation service expects.
- Use `ControllerService` any time you need to push work to worker services.

## Step 4 – Main Service Class

**`services/core/src/your_project_core/your_project_core.py`**

```python
import asyncio
import logging
from enum import Enum
from typing import Optional, Dict

from experimance_common.base_service import BaseService
from experimance_common.zmq.services import ControllerService
from experimance_common.schemas import (
    YourNewMessage, ImageReady, RenderRequest, DisplayMedia,
    ContentType, MessageType,
)
from .config import YourProjectCoreConfig

logger = logging.getLogger(__name__)

class CoreState(Enum):
    """Application-level states (distinct from BaseService lifecycle states)."""
    IDLE = "idle"
    PROCESSING = "processing"
    GENERATING = "generating"

class YourProjectCoreService(BaseService):
    def __init__(self, config: YourProjectCoreConfig):
        super().__init__(config)
        self.config = config
        self.core_state = CoreState.IDLE
        self.zmq_service: ControllerService = None  # type: ignore

    async def start(self):
        self.zmq_service = ControllerService(self.config.zmq)
        self.zmq_service.add_message_handler(
            MessageType.YOUR_NEW_MESSAGE, self._handle_your_message
        )
        self.zmq_service.add_response_handler(self._handle_worker_response)
        self.add_task(self._periodic_task())
        await self.zmq_service.start()
        await super().start()  # call LAST

    async def stop(self):
        if self.zmq_service:
            await self.zmq_service.stop()
        await super().stop()

    async def _handle_your_message(self, topic: str, data: Dict):
        try:
            message = YourNewMessage(**data)
            await self._transition_to_state(CoreState.PROCESSING)
            await self._request_image_generation(message.content)
        except Exception as e:
            logger.error(f"Error handling message: {e}")

    async def _handle_worker_response(self, worker_name: str, response_data: Dict):
        if worker_name == "image_server":
            image_ready = ImageReady(**response_data)
            await self._handle_image_ready(image_ready)

    async def _handle_image_ready(self, image_ready: ImageReady):
        display_message = DisplayMedia(
            content_type=ContentType.IMAGE,
            uri=image_ready.uri,
        )
        await self.zmq_service.publish(display_message)
        await self._transition_to_state(CoreState.IDLE)

    async def _request_image_generation(self, prompt: str):
        render_request = RenderRequest(prompt=prompt, width=1920, height=1080)
        await self._transition_to_state(CoreState.GENERATING)
        await self.zmq_service.send_work_to_worker("image_server", render_request)

    async def _transition_to_state(self, new_state: CoreState):
        logger.info(f"State: {self.core_state.value} → {new_state.value}")
        self.core_state = new_state

    async def _periodic_task(self):
        while self.running:
            logger.debug(f"State: {self.core_state.value}")
            await self._sleep_if_running(30.0)

async def run_your_project_core_service(config_path=None, args=None):
    config = YourProjectCoreConfig.from_overrides(config_file=config_path, args=args)
    service = YourProjectCoreService(config)
    await service.start()
    await service.run()
```

**Important notes:**
- Use `self.core_state` for application state; `self.state` is reserved for `BaseService` lifecycle.
- Register message handlers **before** calling `await self.zmq_service.start()`.
- Call `await super().start()` **last** in your `start()` method.
- ZMQ message handlers receive `(topic: str, data: Dict)`.
- Use `self.zmq_service: ControllerService = None  # type: ignore` to satisfy type checkers before initialisation.

## Step 5 – Entry Point

**`services/core/src/your_project_core/__main__.py`**

```python
import asyncio
from experimance_common.cli import run_service_cli
from .your_project_core import run_your_project_core_service
from .config import YourProjectCoreConfig

if __name__ == "__main__":
    asyncio.run(run_service_cli(
        service_name="Your Project Core",
        service_description="Core orchestration service for the Your Project installation",
        config_class=YourProjectCoreConfig,
        service_runner=run_your_project_core_service,
    ))
```

**`services/core/src/your_project_core/__init__.py`**

```python
from .your_project_core import YourProjectCoreService, run_your_project_core_service
from .config import YourProjectCoreConfig

__all__ = ["YourProjectCoreService", "YourProjectCoreConfig", "run_your_project_core_service"]
```

## Step 7 – Register in pyproject.toml

In `services/core/pyproject.toml`:

```toml
[project.scripts]
your-project-core = "your_project_core.__main__:main"
```

Then run `uv sync` from `services/core` to register the new entry point.

```bash
# 1. Verify import

  "from your_project_core import YourProjectCoreService; print('OK')"
```

## Common Pitfalls

| Symptom | Cause | Fix |
|---|---|---|
| `ImportError: cannot import name 'YourMessage'` | Schema not loaded | Check `experimance_common.schemas` dynamic loading; ensure `PROJECT_ENV` is set |
| `KeyError: <YourState.IDLE: 'idle'>` | Used `self.state` instead of `self.core_state` | Use `self.core_state` for app state |
| `AttributeError: 'NoneType' … 'send_work_to_worker'` | `zmq_service` not initialised | Initialise in `start()`, not `__init__()` |
| `ImportError: cannot import name 'load_config'` | Wrong config helper | Use `YourConfigClass.from_overrides()` |
| Cannot add new message types to enum | Python enums are not extendable | Redeclare the entire `MessageType` enum |
